# Skill File Schema

A strong `SKILL.md` should include:

1. YAML frontmatter with `name` and `description`.
2. H1 title.
3. When to use.
4. Inputs.
5. Outputs.
6. Workflow / Steps.
7. Rules or contracts.
8. Handoff to other skills.
9. Non-negotiables.
10. Quality checklist.
11. Optional test prompts.

## Naming Rules

- Use lowercase kebab-case.
- Prefer domain + action: `prd-authoring`, `citation-bibliography`.
- Custom user skills should start with `custom-` or `user-`.

## Description Rules

The description should say:

- Domain.
- Trigger.
- Main capabilities.
- Deliverable type.
