# Timeline Contract Reference

The frontend timeline should explain the real workflow, not a decorative loading
sequence.

## Required Nodes

1. Read user question and uploads.
2. Chief creates planning contract.
3. Source files are parsed and grounded.
4. Reference examples are analyzed for structure.
5. Skills are loaded and routed.
6. Outline is generated.
7. Sections are drafted.
8. Tables, figures, and citations are inserted.
9. QA repairs issues.
10. Preview and export are generated from the same content.

## Node Detail Rules

- Each node should show owner, skill, input, output, and validation.
- Skill nodes should be clickable to inspect `SKILL.md`.
- File nodes should indicate whether a file is evidence or style-only.
- Repair nodes should show what was detected and how it was fixed.
