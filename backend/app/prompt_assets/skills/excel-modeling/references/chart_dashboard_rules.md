# Chart And Dashboard Rules

## Chart Selection

| Question | Preferred Chart |
|----------|-----------------|
| Trend over time | Line or area chart |
| Category comparison | Bar chart |
| Composition | Stacked bar or 100% stacked bar |
| Distribution | Histogram or box-style summary |
| Relationship | Scatter plot |
| Actual vs target | Bullet chart, bar plus target line, or variance table |
| Scenario comparison | Small table plus line/bar chart |

## Dashboard Layout

- Start with 3-6 KPI cards only when they answer the user's main question.
- Place filters or assumptions near the outputs they affect.
- Put detailed tables below summary visuals.
- Keep one dominant question per dashboard sheet.
- Avoid tiny chart labels and crowded legends.

## Chart Requirements

- Include title, unit, time period, and source range.
- Use consistent colors for the same entities across charts.
- Do not use pie charts for many categories.
- Sort bars intentionally when ranking matters.
- Keep chart data ranges visible or documented.

## PPT/Word Handoff

- Provide concise chart takeaway text.
- Preserve source ranges and assumptions for citations.
- Prefer editable chart specs for downstream export.
