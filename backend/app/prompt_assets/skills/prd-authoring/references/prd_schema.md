# PRD Schema

## Required Sections

1. Background and objective.
2. Users and scenarios.
3. Success metrics.
4. Scope and non-scope.
5. Functional requirements.
6. User stories.
7. Interaction/process flows.
8. Data and analytics.
9. Non-functional requirements.
10. Dependencies and risks.
11. Release plan.
12. Acceptance criteria.

## Requirement Format

```json
{
  "id": "FR-001",
  "priority": "P0|P1|P2",
  "user_story": "",
  "description": "",
  "acceptance_criteria": [],
  "dependencies": [],
  "risks": []
}
```
