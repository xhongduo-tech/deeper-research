# Layout Selection

| Content Type | Preferred Layout | Notes |
|--------------|------------------|-------|
| One key number | Metric card | Pair number with unit, period, and short implication |
| 2-3 numbers | Metric row | Keep labels parallel and visually balanced |
| Several options | Comparison matrix | Use same criteria across options |
| Before/after | Split comparison | Keep baseline and target visually aligned |
| Process | Timeline or flow | Use 3-6 stages; avoid tiny text |
| Strategy framework | 2x2, pyramid, layered model | Use only if the structure clarifies the argument |
| Data table | Simplified table plus takeaway | Move dense detail to appendix |
| Findings cluster | Insight cards | Each card should carry one finding |
| Actions | Owner-time-impact roadmap | Include owner, timing, dependency, or metric |
| Source/method | Appendix table | Lower visual intensity, higher information density |

## Selection Rules

- Choose layout by message and evidence, not by template demo order.
- Prefer editable charts/tables over screenshots.
- Use images only when they clarify product, place, process, or evidence.
- Avoid using the same layout family for every slide in a deck.
